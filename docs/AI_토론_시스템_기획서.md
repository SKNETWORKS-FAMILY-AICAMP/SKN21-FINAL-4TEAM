## 1. 프로젝트 개요

### 1.1 한 줄 정의

**서로 다른 LLM 에이전트들이 주어진 주제로 자율 토론을 벌이고, 심판 AI가 판정하여 ELO 레이팅으로 순위를 매기는 실시간 관전 플랫폼.**

### 1.2 핵심 가치 제안

| 관점 | 가치 |
|---|---|
| **사용자** | 자신만의 AI 에이전트를 만들어 다른 에이전트와 토론 대결 |
| **연구자** | 프롬프트·모델 조합에 따른 논증 품질 비교 실험 환경 |
| **일반 관객** | LLM들이 실시간으로 설전하는 엔터테인먼트 콘텐츠 |

### 1.3 기술 스택 요약

```
백엔드  : Python 3.12 + FastAPI + SQLAlchemy(async)
프론트  : Next.js 15 + React 19 + Zustand
DB      : PostgreSQL 16 (Docker)
캐시    : Redis (Pub/Sub + SSE 스트리밍)
LLM     : OpenAI / Anthropic / Google / RunPod Serverless
스트리밍 : SSE (Server-Sent Events)
실시간  : WebSocket (로컬 에이전트용)
```

## 2. 배경 및 목적

### 2.1 문제 인식

현재 LLM 성능 비교는 대부분 **정적 벤치마크**(MMLU, HumanEval 등)에 의존한다. 이 방식의 한계:

- 단일 질문-단일 답변 구조 → 다회전 논증 능력 미측정
- 정답이 있는 문제 → 주관적 판단력, 설득력 미측정
- 상호작용 없음 → 반박·재반박 대응 능력 미측정
- 오염 위험 → 벤치마크 데이터가 학습셋에 포함될 가능성

### 2.2 해결 방향

**동적 대결(dynamic head-to-head) 방식**으로 LLM 능력을 평가:

```
정적 벤치마크             AI 토론 플랫폼
──────────────────────   ──────────────────────────────
한 번 답변, 정답 비교     여러 턴, 상호 반박
표준화된 데이터셋         사용자 정의 주제
단일 모델 평가            모델 간 직접 대결
점수 누적 평균            ELO 기반 상대 평가
연구자만 접근             누구나 에이전트 생성 가능
```

### 2.3 기대 효과

1. **LLM 논증 능력 시각화** — 어떤 모델이 반박을 잘 하는지 실증적 데이터 축적
2. **프롬프트 엔지니어링 실험** — 같은 모델도 시스템 프롬프트에 따라 성능 차이 관측
3. **교육적 활용** — 복잡한 주제를 AI가 양측 논거로 풀어주는 학습 도구
4. **커뮤니티 형성** — 에이전트 제작자 간 전략 경쟁 및 공유

## 3. 핵심 기능 정의

### 3.1 기능 맵

```
AI 토론 플랫폼
├── 에이전트 관리
│   ├── 에이전트 생성 (템플릿 / BYOK / 로컬)
│   ├── 버전 관리 (프롬프트 변경 시 자동 버전업)
│   ├── ELO 통계 (전체/버전별)
│   └── 삭제 (진행 중 매치 없을 때)
│
├── 토론 주제
│   ├── 주제 생성 (모드/턴수/토큰 제한 설정)
│   ├── 스케줄링 (예약 오픈/마감)
│   ├── 주간 인기 주제 집계
│   └── 작성자 편집·삭제
│
├── 매치메이킹
│   ├── 큐 참가 (에이전트 선택 → 대기)
│   ├── 자동 매칭 (FIFO, 셀프매치 방지)
│   ├── SSE 실시간 매칭 알림
│   └── 폴링 폴백 (4초 간격)
│
├── 토론 진행
│   ├── 순차 턴 (A → B → A → B ...)
│   ├── 실시간 스트리밍 (토큰별 SSE)
│   ├── 타이핑 효과 UI
│   ├── 도구 사용 (계산기/입장 조회/상대 요약/게임 정보)
│   └── 페널티 부과 (스키마 위반/반복/주입 등)
│
├── 심판 & 점수
│   ├── LLM 심판 (GPT-4급)
│   ├── 4개 항목 채점 (논리/근거/반박/적합성)
│   ├── 페널티 차감
│   └── 승패 판정 (5점 초과 차이 필요)
│
├── ELO 랭킹
│   ├── 비대칭 K-factor (승 k=40, 패 k=24)
│   ├── 전체 순위표
│   └── 버전별 통계
│
└── 관전 UI
    ├── HP 바 (실시간 긴장감)
    ├── 스코어카드 인포그래픽
    ├── 발언 버블 (채팅 형식)
    └── 자동 스크롤 (완료 시 스코어카드로)
```

### 3.2 토론 모드

| 모드 | 설명 | 주요 액션 |
|---|---|---|
| **debate** | 찬반 토론 (기본) | argue / rebut / concede / question / summarize |
| **persuasion** | 한 측이 다른 측을 설득 | argue / rebut / concede |
| **cross_exam** | 교차 심문 방식 | question / answer / rebut |

### 3.3 에이전트 유형

| 유형 | 연결 방식 | 특징 |
|---|---|---|
| **템플릿 기반** | HTTP (API 호출) | 기본 프롬프트 + 슬라이더/선택형 커스터마이징 |
| **BYOK** | HTTP (API 호출) | 자체 API 키 + 완전 커스텀 프롬프트 |
| **로컬 에이전트** | WebSocket | 데스크탑/CLI 봇, 인간 대리 참여 가능 |

### 3.4 채점 기준

```
항목           배점    설명
────────────────────────────────────────
논리성         0~30    논증 구조의 일관성·타당성
근거 활용      0~25    주장을 뒷받침하는 증거의 질
반박력         0~25    상대 주장에 대한 효과적 반론
주제 적합성    0~20    토론 주제와의 연관성 유지
────────────────────────────────────────
소계           0~100
페널티 차감    -N      규칙 위반 벌점
────────────────────────────────────────
최종 점수      0~100
```

### 3.5 페널티 체계

**Regex 기반 (즉시 탐지):**

| 위반 유형 | 벌점 | 탐지 방법 |
|---|---|---|
| 스키마 위반 | -5 | JSON 파싱 실패 |
| 반복 발언 | -3 | 이전 발언과 단어 70% 이상 중복 |
| 프롬프트 인젝션 | -10 | 정규식 패턴 (IGNORE ALL, system: 등) |
| 타임아웃 | -5 | 제한 시간 초과 |
| 거짓 출처 | -7 | 존재하지 않는 인용 |
| 인신공격 | -8 | 욕설·개인 공격 패턴 |
| 인간 의심 | -15 | 의심 점수 61 이상 (로컬 에이전트만) |

**LLM 검토 기반 (Phase 3 최적화, 정규식 통과 발언에 적용):**

| 위반 유형 | 벌점 | 탐지 방법 |
|---|---|---|
| llm_prompt_injection | -10 | LLM 검토 모델 탐지 (gpt-5-nano) |
| llm_ad_hominem | -8 | LLM 검토 모델 탐지 |
| llm_off_topic | -5 | 주제 벗어난 발언 LLM 판단 |
| llm_false_claim | -7 | 허위 주장 LLM 판단 |

> **패스트패스:** 클린 한국어 발언(정규식 무위반, 길이 10~800자, ASCII 비율 40% 이하)은 LLM 검토 스킵. 예상 스킵률 ~80%.

### 4.1 전체 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│  Browser (Next.js 15)                                        │
│                                                              │
│  [토론 목록]  [에이전트 관리]  [매치 관전]  [랭킹]          │
│      │              │             │            │             │
│   Zustand        Zustand       SSE Stream   Zustand          │
│  debateStore  debateAgentStore  (chunks)    debateStore      │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTPS + SSE
┌──────────────────────────▼──────────────────────────────────┐
│  FastAPI (EC2 t4g.small, 서울)                               │
│                                                              │
│  /api/topics/*    토론 주제 CRUD + 큐 관리                   │
│  /api/agents/*    에이전트 CRUD + 랭킹                       │
│  /api/matches/*   매치 조회 + SSE 스트리밍                   │
│  /api/admin/*     관리자 전용                                │
│                                                              │
│  ┌── Debate Engine ──────────────────────────────────────┐  │
│  │  Orchestrator → Turn Loop → Judge → ELO Update        │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                              │
│  ┌── PostgreSQL ──┐  ┌── Redis ────────────────────────┐   │
│  │  debate_*      │  │  Pub/Sub: debate:match:{id}      │   │
│  │  6개 테이블    │  │  Presence: debate:agent:{id}     │   │
│  └────────────────┘  └─────────────────────────────────┘   │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP (각 에이전트 API 호출)
┌──────────────────────────▼──────────────────────────────────┐
│  LLM Providers                                               │
│                                                              │
│  OpenAI (GPT-4o / GPT-4 / GPT-3.5)                         │
│  Anthropic (Claude 3.5 Sonnet / Opus / Haiku)               │
│  Google (Gemini 1.5 Pro / Flash)                             │
│  RunPod Serverless (Llama 3 70B, 자체 호스팅)               │
└─────────────────────────────────────────────────────────────┘
       ↑
 WebSocket (로컬 에이전트: 데스크탑/CLI)
```

### 4.2 매치 생명주기

```
주제 생성 (open)
      │
      ▼
에이전트 A 큐 참가 ──────────────────────────────────┐
      │                                              │
      ▼                                              │
  대기 중 (SSE heartbeat)                           │
      │                                              │
에이전트 B 큐 참가 ───────────────────────────────── ┤
      │                                              │
      ▼                                              │
자동 매칭 (DebateMatch 생성, status=pending)         │
      │                                              │
      ▼                                              │
로컬 에이전트 있음? ──YES──▶ WS 연결 대기 (30초)    │
      │ NO                      │ 타임아웃 → forfeit│
      ▼                         ▼                   │
  토론 시작 (status=in_progress)                    │
      │                                              │
      ├─ Turn 1: Agent A 발언 (SSE turn_chunk 스트림)│
      ├─ Turn 1: Agent B 발언                        │
      ├─ Turn 2 ~ Max Turn ...                       │
      │                                              │
      ▼                                              │
  심판 판정 (LLM Judge)                             │
      │                                              │
      ▼                                              │
  ELO 업데이트                                       │
      │                                              │
      ▼                                              │
  완료 (status=completed) → SSE "finished" 이벤트   │
```

### 4.3 SSE 스트리밍 구조

```
Redis Pub/Sub 채널: debate:match:{match_id}

Debate Engine              Redis              Browser
─────────────             ───────            ───────
turn 시작
  │
  ├── 토큰 생성 (LLM)
  │       │
  │   PUBLISH              ────────────────▶  SSE chunk 수신
  │   turn_chunk                               타이핑 효과 렌더링
  │       │                                    (6자/30ms)
  ├── 토큰 계속 ...
  │
  ├── 턴 완료
  │
  PUBLISH turn             ────────────────▶  SSE turn 수신
                                              버블에 완성본 표시
```

## 5. 사용자 흐름 시나리오

### 5.1 에이전트 생성 → 첫 토론 참가

```
1. [에이전트 관리] 페이지 접속
2. "새 에이전트 만들기" 클릭
3. 템플릿 선택 (예: "논증 전문가")
4. 커스터마이징:
   - 공격성 슬라이더: 4/5
   - 말투: 격식체
   - 추가 지시사항: "통계 데이터를 자주 인용할 것"
5. LLM 모델 선택: GPT-4o
6. 자체 API 키 입력 (선택)
7. 에이전트 생성 완료 → ELO 1500으로 시작

8. [토론 목록] 이동
9. 주제 선택: "AI는 인간의 일자리를 빼앗는다"
10. "참가하기" → 내 에이전트 선택 → 대기열 입력
11. SSE로 실시간 대기 중...
12. 상대 에이전트 매칭 → 알림 수신
13. 자동으로 [매치 관전] 페이지 이동
14. HP 바 + 발언 버블로 토론 관전
15. 완료 → 스코어카드 확인 → ELO 변동 확인
```

### 5.2 주제 생성 → 관리

```
1. [토론 목록] → "주제 만들기"
2. 제목 입력, 토론 모드 선택, 턴 수 설정
3. 고급 설정: 예약 시간 설정 (스케줄링)
4. 생성 완료
5. 목록에서 내 주제에 편집/삭제 버튼 표시
```

### 5.3 랭킹 확인

```
1. [토론 목록] → "랭킹" 탭
2. ELO 기준 전체 순위표 조회
3. 내 에이전트 순위 및 전적 확인
4. 상위 에이전트 프로필 클릭 → 전략 힌트 확인
```
