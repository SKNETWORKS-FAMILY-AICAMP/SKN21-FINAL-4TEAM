## 1. 로컬 개발 환경 구성

### 필수 소프트웨어

| 도구 | 버전 | 용도 |
|---|---|---|
| Python | 3.10+ | 백엔드 |
| Node.js | 20+ | 프론트엔드 |
| PostgreSQL | 16 + pgvector | DB (Docker 권장) |
| Redis | 7+ | 캐시/세션 (Docker 권장) |
| Docker Compose | v2 | DB/Redis 일괄 기동 |

### 초기 세팅

```bash
# 1. DB + Redis 기동
docker compose up db redis -d

# 2. 백엔드
cd backend
python -m venv .venv && .venv\Scripts\activate   # Windows
pip install -r requirements-dev.txt                # 개발/테스트 의존성 포함
alembic upgrade head                              # DB 마이그레이션 (32 테이블)
uvicorn app.main:app --reload --port 8000

# 3. 프론트엔드
cd frontend
npm install
npm run dev   # http://localhost:3000
```

### 환경 변수

`backend/.env`에서 관리. 주요 항목:

| 변수 | 설명 | 필수 |
|---|---|---|
| `DATABASE_URL` | PostgreSQL 접속 URL (asyncpg) | O |
| `REDIS_URL` | Redis 접속 URL | O |
| `SECRET_KEY` | JWT 서명 키 (프로덕션에서 강제) | O |
| `OPENAI_API_KEY` | OpenAI API 키 (해당 모델 사용 시) | - |
| `ANTHROPIC_API_KEY` | Anthropic API 키 | - |
| `PIPELINE_LAZY_LOAD` | `True`=첫 호출 시 모델 로드, `False`=서버 시작 시 로드 | - |
| `TTS_PROVIDER` | TTS 프로바이더 (`openai` \| `elevenlabs`) | - |
| `TTS_ENABLED` | TTS 기능 활성화 (기본: `true`) | - |
| `IMAGE_GEN_PROVIDER` | 이미지 생성 프로바이더 (`openai` \| `stability`) | - |
| `IMAGE_GEN_ENABLED` | 이미지 생성 기능 활성화 (기본: `true`) | - |

> `.env`에는 Docker Compose 자격증명도 포함되어 있으나, `Settings(extra="ignore")`로 백엔드에서 무시됩니다.

## 3. 백엔드 수정 가이드

### 3.1 계층 구조 이해

```
요청 → API 라우터 (api/) → 서비스 (services/) → DB/외부 API
         │                      │
         ├── Pydantic 검증      ├── 비즈니스 로직
         ├── 인증/권한 (deps)    ├── ORM 쿼리
         └── 응답 직렬화        └── 파이프라인 호출
```

**규칙:**
- 라우터(`api/`)에는 비즈니스 로직을 넣지 않는다. 서비스 호출만 한다.
- 서비스(`services/`)에서 DB 쿼리, 외부 API 호출, 정책 검증을 처리한다.
- HTTP 예외(`HTTPException`)는 서비스에서 직접 raise 가능하다.

### 3.2 새 API 엔드포인트 추가

```python
# 1. 스키마 정의 — backend/app/schemas/my_feature.py
from pydantic import BaseModel, Field

class MyFeatureCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    description: str = Field(..., max_length=2000)

# 2. 서비스 로직 — backend/app/services/my_feature_service.py
class MyFeatureService:
    def __init__(self, db: AsyncSession):
        self.db = db

    async def create(self, data: MyFeatureCreate, user: User) -> MyModel:
        # 비즈니스 로직 + DB 저장
        ...

# 3. 라우터 — backend/app/api/my_feature.py
from app.core.deps import get_current_user

router = APIRouter()

@router.post("/", status_code=201)
async def create_feature(
    data: MyFeatureCreate,
    user: User = Depends(get_current_user),  # 인증 필수
    db: AsyncSession = Depends(get_db),
):
    service = MyFeatureService(db)
    return await service.create(data, user)

# 4. 라우터 등록 — backend/app/main.py
from app.api import my_feature
app.include_router(my_feature.router, prefix="/api/my-feature", tags=["my-feature"])
```

### 3.3 인증/권한 의존성

| 의존성 | 용도 | 사용 예 |
|---|---|---|
| `get_current_user` | 로그인 사용자 반환 | 모든 인증 필요 API |
| `require_admin` | admin 또는 superadmin만 통과 | 관리자 API |
| `require_superadmin` | superadmin만 통과 | 시스템 설정, 모델 관리 |
| `require_adult_verified` | 성인인증 완료 사용자만 통과 | 18+ 콘텐츠 API |

```python
# 관리자 전용 엔드포인트
@router.get("/")
async def admin_only(admin: User = Depends(require_admin)):
    ...

# 성인인증 필수 엔드포인트
@router.get("/adult-content")
async def adult_only(user: User = Depends(require_adult_verified)):
    ...
```

```ts
import { create } from 'zustand';
import { api } from '@/lib/api';

type MyFeatureState = {
  items: Item[];
  loading: boolean;
  fetchItems: () => Promise<void>;
};

export const useMyFeatureStore = create<MyFeatureState>((set) => ({
  items: [],
  loading: false,
  fetchItems: async () => {
    set({ loading: true });
    try {
      const data = await api.get<{ items: Item[] }>('/my-feature');
      set({ items: data.items });
    } finally {
      set({ loading: false });
    }
  },
}));
```

```python
def compile(self, persona, lorebook_entries, session_summary, recent_messages):
    messages = []
```

### 5.3 LLM 모델 추가

새 LLM 프로바이더를 추가하려면:

1. `services/inference_client.py`에 `_call_새프로바이더()` + `_stream_새프로바이더()` 메서드 추가
2. `generate()`와 `generate_stream()`의 provider 분기에 추가
3. DB `llm_models` 테이블에 새 모델 레코드 INSERT

```python
# services/inference_client.py
async def generate(self, model: LLMModel, messages: list[dict]) -> dict:
    match model.provider:
        case "openai":    return await self._call_openai(model, messages)
        case "anthropic": return await self._call_anthropic(model, messages)
        case "google":    return await self._call_google(model, messages)
        case "runpod":    return await self._call_runpod(model, messages)
        case "새프로바이더": return await self._call_new_provider(model, messages)
        case _:           raise ValueError(f"Unknown provider: {model.provider}")
```

### 7.1 아키텍처


```python
class MyAnalyzer:

    def __init__(self):

        if self.model is None:
            self.model = load_my_model()

    def analyze(self, text: str) -> dict:
        self._ensure_loaded()
        return self.model.predict(text)
```

## 11. 코드 리뷰 수정 사항

최근 코드 리뷰에서 적용된 주요 개선 사항입니다. 새 코드 작성 시 반드시 따르세요.

### 11.1 AbortController 지원

`api.ts`에 `signal`(AbortController) 지원이 추가되었습니다. 모든 `useEffect` 내 API 호출에 AbortController를 적용하여 컴포넌트 언마운트 시 불필요한 요청을 취소하세요.

```tsx
useEffect(() => {
  const controller = new AbortController();
  api.get('/my-endpoint', { signal: controller.signal })
    .then(setData)
    .catch((err) => {
      if (err.name !== 'AbortError') toast.error('로드 실패');
    });
  return () => controller.abort();
}, []);
```

### 11.2 모달 포커스 트랩

모달 컴포넌트에는 포커스 트랩 패턴을 적용해야 합니다. 모달이 열려 있는 동안 Tab 키가 모달 내부 요소만 순회하도록 구현하세요. 기존 모달 컴포넌트(`AdultVerifyModal`, `AgeGateModal` 등)에 이 패턴이 적용되어 있으니 참고하세요.

### 11.3 아이콘 버튼 접근성

아이콘 전용 버튼에는 반드시 `aria-label`을 추가해야 합니다.

```tsx
// Good
<button aria-label="삭제" onClick={handleDelete}>
  <TrashIcon className="w-5 h-5" />
</button>

// Bad — 스크린 리더가 버튼 용도를 알 수 없음
<button onClick={handleDelete}>
  <TrashIcon className="w-5 h-5" />
```
