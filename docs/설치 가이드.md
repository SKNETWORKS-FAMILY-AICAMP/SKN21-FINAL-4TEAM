## 3. 인프라 구성 (Docker)

PostgreSQL(+pgvector)과 Redis를 Docker로 실행합니다.

### 방법 A: DB + Redis만 실행 (로컬 개발 권장)

```bash
# 프로젝트 루트에서 실행
docker compose up -d db redis
```

동작 확인:

```bash
# PostgreSQL 접속 확인
docker exec -it chatbot-db psql -U chatbot -d chatbot -c "SELECT 1;"

# Redis 접속 확인
docker exec -it chatbot-redis redis-cli -a redis_dev_pw_2026 ping
# → PONG
```

### 방법 B: 모니터링 포함 실행

```bash
docker compose up -d db redis prometheus grafana langfuse
```

| 서비스 | URL | 비고 |
|---|---|---|
| PostgreSQL | `localhost:5432` | user: chatbot / pw: chatbot_dev_pw_2026 |
| Redis | `localhost:6379` | pw: redis_dev_pw_2026 |
| Prometheus | http://localhost:9090 | 메트릭 조회 |
| Grafana | http://localhost:3002 | admin / grafana_dev_pw_2026 |
| Langfuse | http://localhost:3001 | LLM 트레이싱 |

### 테스트용 DB 생성

통합 테스트는 `chatbot_test` 데이터베이스를 사용합니다. 최초 1회 생성이 필요합니다:

```bash
docker exec -it chatbot-db psql -U chatbot -d chatbot -c "CREATE DATABASE chatbot_test OWNER chatbot;"
docker exec -it chatbot-db psql -U chatbot -d chatbot_test -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

## 4. 백엔드 설정

### 가상환경 생성 & 의존성 설치

```bash
cd backend

# 가상환경 생성
python -m venv .venv

# 가상환경 활성화
# Windows (PowerShell)
.venv\Scripts\Activate.ps1
# Windows (CMD)
.venv\Scripts\activate.bat
# macOS/Linux
source .venv/bin/activate

# 의존성 설치 (개발용 — 테스트/린트 포함)
pip install -r requirements-dev.txt
```

> **참고:** 운영 환경에서는 `pip install -r requirements.txt` (테스트/린트 제외)

### GPU가 없는 경우

ML 파이프라인(`torch`, `transformers`)은 GPU 없이도 CPU 모드로 동작합니다. 다만 모델 로딩이 느릴 수 있으므로, `.env`에서 지연 로딩을 활성화하세요:

```env
PIPELINE_LAZY_LOAD=true
PIPELINE_DEVICE=cpu
```

### 디렉토리 구조 확인

```
backend/
├── .env                 ← 환경변수 (git 추적 안 됨)
├── requirements.txt     ← 운영 의존성
├── requirements-dev.txt ← 개발 의존성 (테스트/린트)
├── pyproject.toml       ← ruff/pytest 설정
├── alembic.ini          ← 마이그레이션 설정
├── alembic/             ← 마이그레이션 파일
├── app/                 ← 애플리케이션 소스
└── tests/               ← 테스트
```

### 의존성 설치

```bash
cd frontend

```

### 주요 설정 파일

| 파일 | 역할 |
|---|---|
| `next.config.js` | API 프록시 (`/api/*` → `localhost:8000`), 이미지 도메인, 번들 최적화 |
| `tailwind.config.ts` | 디자인 토큰 (다크 테마 색상, 폰트, 그림자) |

### API 프록시

프론트엔드의 `/api/*` 요청은 `next.config.js`의 rewrites 설정에 의해 `http://localhost:8000/api/*`로 프록시됩니다. 별도의 CORS 설정이나 API URL 환경변수가 필요 없습니다.

## 9. Docker Compose 전체 실행

로컬 개발이 아닌 **통합 테스트** 또는 **스테이징** 목적으로 전체 스택을 Docker로 실행할 수 있습니다.

```bash
# .env.example을 프로젝트 루트 .env로도 복사
cp .env.example .env

# 전체 서비스 빌드 & 실행
docker compose up -d --build

# 로그 확인
docker compose logs -f backend frontend

# 종료
docker compose down
```

| 서비스 | URL |
|---|---|
| 프론트엔드 | http://localhost:3000 |
| 백엔드 API | http://localhost:8000 |
| Nginx (HTTPS) | https://localhost (SSL 인증서 필요) |
| Grafana | http://localhost:3002 |
| Langfuse | http://localhost:3001 |
| Prometheus | http://localhost:9090 |

> **주의:** Docker Compose로 전체 실행 시 `backend/.env`의 DB/Redis 호스트가 `localhost`이면 컨테이너 간 통신이 안 됩니다. Docker 네트워크에서는 서비스 이름(`db`, `redis`)을 사용해야 합니다. 로컬 개발(방법 A)에서는 DB/Redis만 Docker, 백엔드/프론트엔드는 호스트에서 실행하므로 `localhost`가 맞습니다.

## 10. GPU 파이프라인 설정 (선택)

감정 분석(KcELECTRA), 임베딩(BGE-M3), 리랭커(bge-reranker-v2-m3) 파이프라인은 GPU에서 훨씬 빠르게 동작합니다.

### CUDA + PyTorch 설치

```bash
# 기존 torch 제거 후 CUDA 버전에 맞게 재설치
pip uninstall torch -y
pip install torch --index-url https://download.pytorch.org/whl/cu121
# CUDA 12.8인 경우: cu128 사용
```

### .env 설정

```env
PIPELINE_DEVICE=cuda         # GPU 사용
PIPELINE_LAZY_LOAD=false     # 서버 기동 시 미리 로드 (첫 요청 지연 방지)
```

### 모델 다운로드

첫 실행 시 HuggingFace에서 자동 다운로드됩니다 (~3GB):

| 모델 | 용도 | 크기 |
|---|---|---|
| `searle-j/kote_for_easygoing_people` | 감정 분석 (43 감정) | ~500MB |
| `BAAI/bge-m3` | 임베딩 (1024차원) | ~2.2GB |
| `BAAI/bge-reranker-v2-m3` | 리랭킹 | ~560MB |

> **GPU 메모리:** 3개 모델 동시 로드 시 약 6GB VRAM 필요. 8GB GPU(RTX 3060/4060/5060)면 충분합니다.

## 11. 외부 서비스 연동 (선택)

### RunPod (기본 LLM 추론)

1. [runpod.io](https://runpod.io) 계정 생성
2. Serverless Endpoint 생성 (Llama 3 70B)
3. `.env`에 설정:
   ```env
   RUNPOD_API_KEY=rp_xxxxxxxxxxxx
   RUNPOD_ENDPOINT_ID=xxxxxxxxxx
   ```

### OpenAI / Anthropic / Google (선택적 LLM)

사용자가 모델을 선택할 수 있도록 외부 LLM API 키를 설정합니다:

```env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_API_KEY=AIza...
```

> 키를 설정하지 않아도 서버는 정상 기동됩니다. 해당 모델 선택 시에만 에러가 발생합니다.

### TTS (음성 합성)

텍스트 음성 변환(TTS) 기능을 사용하려면 다음 환경 변수를 설정합니다:

```env
# OpenAI TTS (기본 프로바이더)
TTS_PROVIDER=openai              # openai | elevenlabs
TTS_DEFAULT_VOICE=alloy          # alloy, echo, fable, onyx, nova, shimmer
TTS_OUTPUT_FORMAT=mp3
TTS_ENABLED=true

# ElevenLabs TTS (선택)
ELEVENLABS_API_KEY=...
ELEVENLABS_DEFAULT_VOICE_ID=...
```

> OpenAI TTS는 `OPENAI_API_KEY`를 공유합니다. 별도 키 설정 불필요.

### 이미지 생성 (AI Image Generation)

AI 이미지 생성 기능을 사용하려면:

```env
# OpenAI DALL-E 3 (기본 프로바이더)
IMAGE_GEN_PROVIDER=openai        # openai | stability
IMAGE_GEN_DEFAULT_STYLE=anime
IMAGE_GEN_ENABLED=true

# Stability AI (선택)
STABILITY_API_KEY=sk-...
```

> OpenAI 이미지 생성은 `OPENAI_API_KEY`를 공유합니다.

### Langfuse (LLM 관측성)

```env
LANGFUSE_PUBLIC_KEY=pk-...
LANGFUSE_SECRET_KEY=sk-...
LANGFUSE_HOST=http://localhost:3001
```

### Sentry (에러 모니터링)

```env
SENTRY_DSN=https://xxxx@sentry.io/xxxx
```

## 12. 트러블슈팅

### PostgreSQL 연결 실패

```
sqlalchemy.exc.OperationalError: connection refused
```

**원인:** Docker의 PostgreSQL이 실행 중이지 않음.

```bash
docker compose up -d db
docker compose logs db     # 로그 확인
```

### pgvector 확장 에러

```
relation "vector" does not exist
```

**원인:** pgvector 확장이 설치되지 않음. `pgvector/pgvector:pg16` 이미지를 사용 중인지 확인:

```bash
docker exec -it chatbot-db psql -U chatbot -d chatbot -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

### Redis 연결 실패

```
redis.exceptions.ConnectionError: Error while reading from socket
```

**원인:** 로컬 개발에서 Redis 비밀번호 불일치.
- 로컬 개발 (DB/Redis만 Docker): `.env`의 `REDIS_URL`에 비밀번호를 **넣지 않음** (`redis://localhost:6379/0`)
- Docker Compose로 Redis를 띄웠다면 비밀번호 필요: `redis://:redis_dev_pw_2026@localhost:6379/0`

### 테스트 DB 에러

```
asyncpg.InvalidCatalogNameError: database "chatbot_test" does not exist
```

**해결:**

```bash
docker exec -it chatbot-db psql -U chatbot -d chatbot -c "CREATE DATABASE chatbot_test OWNER chatbot;"
docker exec -it chatbot-db psql -U chatbot -d chatbot_test -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

### Windows에서 kiwipiepy 설치 실패

```
error: Microsoft Visual C++ 14.0 or greater is required
```

**해결:** [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) 설치 후 "C++ 빌드 도구" 워크로드 선택.

### torch 설치 시 CUDA 버전 불일치

```
CUDA error: no kernel image is available for execution
```

**해결:** GPU에 맞는 CUDA 버전으로 torch 재설치:

```bash
# CUDA 버전 확인
```

## 빠른 시작 요약

```bash
# 1. 저장소 클론
git clone <repository-url> && cd Project_New

# 2. 환경변수
cp .env.example backend/.env
# → backend/.env에서 SECRET_KEY 변경

# 3. DB + Redis 실행
docker compose up -d db redis

# 4. 테스트 DB 생성
docker exec -it chatbot-db psql -U chatbot -d chatbot \
  -c "CREATE DATABASE chatbot_test OWNER chatbot;"
docker exec -it chatbot-db psql -U chatbot -d chatbot_test \
  -c "CREATE EXTENSION IF NOT EXISTS vector;"

# 5. 백엔드 설정
cd backend
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements-dev.txt
alembic upgrade head
uvicorn app.main:app --reload --port 8000

# 6. 프론트엔드 설정 (새 터미널)
cd frontend
npm install
npm run dev

# 7. 접속
# 프론트엔드: http://localhost:3000
# 백엔드 API: http://localhost:8000/docs (Swagger UI)
```
