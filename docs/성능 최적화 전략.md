# 성능 최적화 전략

## 1. 개요 & 성능 목표

### 아키텍처 요약

```
사용자 ─── HTTPS/SSE ──→ Nginx (HTTP/2) ──→ FastAPI (EC2 서울 t4g.small)
                                                    │
                            ┌───────────────────────┤
                            │                       │
                      PostgreSQL 16            Redis 7
                      + pgvector               (256MB LRU)
                            │
                            │ HTTP (~150ms RTT)
                            ▼
                      RunPod Serverless (미국)
                      SGLang + Llama 3 70B (AWQ 4-bit)
                      + 외부 LLM API (OpenAI/Anthropic/Google)
```

- **단계:** 프로토타입 (동시 접속 10명 이하)
- **서버:** EC2 t4g.small (서울) — 제어 플레인 + DB + Redis
- **GPU 추론:** RunPod Serverless (미국) — SGLang 엔진, 초 단위 과금
- **월 예상 비용:** ~$130 (EC2 ~$15 + RunPod ~$114) + LLM API 비용

## 2. 백엔드 성능 전략

### 2.1 비동기 아키텍처

전체 I/O 경로를 `async/await`로 통일하여 이벤트 루프 블로킹을 방지한다.

| 계층 | 라이브러리 | 비동기 방식 |
|---|---|---|
| HTTP 프레임워크 | FastAPI + Uvicorn | ASGI, `--workers 2` |
| DB 드라이버 | asyncpg (SQLAlchemy async) | `create_async_engine` |
| Redis | redis.asyncio | `Redis.from_url(decode_responses=True)` |
| HTTP 클라이언트 | httpx | `httpx.AsyncClient(timeout=...)` |

```python
# backend/app/core/database.py
engine = create_async_engine(
    settings.database_url,
    pool_size=5,
    max_overflow=10,
)
async_session = async_sessionmaker(engine, expire_on_commit=False)
```

- `expire_on_commit=False` — 커밋 후 객체 재로딩 방지 (불필요한 SELECT 제거)
- sync 함수에서 `asyncio.run()` 직접 호출 금지 — 이벤트 루프 충돌 방지

### 2.2 SSE 스트리밍

LLM 응답을 `AsyncGenerator` + `StreamingResponse`로 청크 단위 전송한다. 서울↔미국 RTT ~150ms를 SSE 스트리밍으로 체감 지연 상쇄.

```python
# backend/app/services/chat_service.py
async def send_message_stream(self, session_id, user, content) -> AsyncGenerator[str, None]:
    usage_out: dict = {}  # Mutable dict로 스트리밍 중 토큰 수 캡처
    async for chunk in self.inference.generate_stream(llm_model, prompt_messages, usage_out=usage_out):
        full_response.append(chunk)
        yield chunk
    # 스트리밍 완료 후 usage_out에서 토큰 수 추출
    input_tokens = usage_out.get("input_tokens", 0)
    output_tokens = usage_out.get("output_tokens", 0)
```

**토큰 추정 폴백:** API가 토큰 수를 반환하지 않는 경우 `len(text) // 3`으로 추정.

**Nginx SSE 설정:**

```nginx
# infra/nginx/nginx.conf
location /api/ {
    proxy_buffering off;           # 청크 즉시 전달 (버퍼링 비활성화)
    proxy_cache off;               # SSE 캐싱 방지
    chunked_transfer_encoding off; # 프레이밍 오버헤드 제거
    proxy_read_timeout 300s;       # 5분 롱 커넥션 허용
    proxy_set_header Connection '';
}
```

```python
def analyze_batch(self, texts: list[str], top_k=5, threshold=0.3) -> list[list[dict]]:
    inputs = self.tokenizer(texts, return_tensors="pt", truncation=True,
                            max_length=512, padding=True)
    with torch.no_grad():
        outputs = self.model(**inputs)

# backend/app/pipeline/embedding.py
def embed_batch(self, texts: list[str]) -> list[list[float]]:
    inputs = self.tokenizer(texts, return_tensors="pt", truncation=True,
                            max_length=8192, padding=True)
    with torch.no_grad():
        outputs = self.model(**inputs)
        embeddings = outputs.last_hidden_state[:, 0, :]      # CLS 토큰
        embeddings = torch.nn.functional.normalize(embeddings, p=2, dim=1)  # L2 정규화
```

### 2.6 RAG 검색 최적화

**2단계 검색 파이프라인 (Over-fetch + Rerank):**

```python
# backend/app/services/rag_service.py
async def retrieve(self, query, top_k=5, over_fetch_factor=3):
    query_vec = self.embedding.embed(query)  # 1024차원 BGE-M3

    # Stage 1: pgvector 코사인 거리 — 상위 15개 후보 (top_k × 3)
    fetch_k = top_k * over_fetch_factor
    stmt = (
        select(EpisodeEmbedding)
        .order_by(EpisodeEmbedding.embedding.cosine_distance(query_vec))
        .limit(fetch_k)
    )

    # Stage 2: 교차 인코더 리랭킹 — 15개 → 5개 정제
    reranked = self.reranker.rerank(query, candidate_texts, top_k=top_k)
```

| 단계 | 방식 | 후보 수 | 속도 |
|---|---|---|---|
| 1단계 | pgvector HNSW (ANN) | 15개 | ~10ms |
| 2단계 | bge-reranker-v2-m3 (교차 인코더) | 5개 | ~50ms |

### 2.8 배치 처리 & 프리컴퓨트

**AsyncIO 큐 기반 배치 스케줄러:**

```python
# backend/app/services/batch_scheduler.py
class BatchScheduler:
    def __init__(self):
        self._queue: asyncio.Queue[str] = asyncio.Queue()
        self._worker_task: asyncio.Task | None = None

    async def _worker_loop(self):
        while self._running:
            try:
                job_id = await asyncio.wait_for(self._queue.get(), timeout=5.0)
                await self._execute_job(job_id)
            except TimeoutError:
                continue  # 큐 비어있으면 5초 대기 후 재시도
```

**청크 병합 전략:**

```python
# backend/app/pipeline/batch.py
@staticmethod
def _merge_sentences_to_chunks(sentences: list[str], max_length: int = 500) -> list[str]:
    """짧은 문장들을 500자 청크로 병합하여 임베딩 배치 크기 최적화."""
    chunks, current, current_len = [], [], 0
    for sent in sentences:
        if current_len + len(sent) > max_length and current:
            chunks.append(" ".join(current))
            current, current_len = [], 0
        current.append(sent)
        current_len += len(sent)
```

**배치 처리 흐름:**

1. 에피소드 텍스트 → 한국어 NLP 문장 분리
2. 문장 → 500자 청크 병합
3. 청크 → 임베딩 배치 생성 (BGE-M3)
4. 텍스트 → 감정 분석 배치 (KcELECTRA)
5. 결과 → DB 일괄 삽입

```typescript
  if (!trimmed || disabled) return;
  onSend(trimmed);
  setText('');
  inputRef.current?.focus();
}, [text, disabled, onSend]);

const handleKeyDown = useCallback((e: React.KeyboardEvent) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    handleSubmit();
  }
}, [handleSubmit]);
```

### 3.4 SSE 스트리밍 UX

**ReadableStream 수동 파싱:**

```typescript
// frontend/src/lib/sse.ts
const decoder = new TextDecoder();
let buffer = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  buffer += decoder.decode(value, { stream: true });  // 멀티바이트 UTF-8 처리
  const lines = buffer.split('\n');

    if (trimmed.startsWith('data: ')) {

}
```
