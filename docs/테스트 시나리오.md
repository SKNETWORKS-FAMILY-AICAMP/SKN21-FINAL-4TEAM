## 1. 랜딩/인증 페이지

**URL:** `/`
**API:** `POST /auth/login`, `POST /auth/register`, `GET /auth/check-nickname`, `GET /auth/me`, `PUT /auth/me`

### TC-1.1 로그인 성공

| 항목 | 내용 |
|------|------|
| [P] | 등록된 계정이 존재 (예: nickname=`testuser`, password=`Test1234!`) |
| [S] | 1. 로그인 탭 선택 → 2. 닉네임/비밀번호 입력 → 3. "로그인" 클릭 |
| [E] | `/personas` 페이지로 리다이렉트, 사이드바에 닉네임 표시, localStorage에 token 저장 |

### TC-1.2 로그인 실패 — 잘못된 비밀번호

| 항목 | 내용 |
|------|------|
| [P] | 등록된 계정 존재 |
| [S] | 닉네임 정확 + 비밀번호 틀리게 입력 → "로그인" 클릭 |
| [E] | 에러 토스트 "닉네임 또는 비밀번호가 올바르지 않습니다" 표시, 페이지 유지 |

### TC-1.3 로그인 실패 — 존재하지 않는 계정

| 항목 | 내용 |
|------|------|
| [P] | 해당 닉네임 미등록 |
| [S] | 존재하지 않는 닉네임 입력 → "로그인" 클릭 |
| [E] | 에러 토스트 표시, 401 응답 |

### TC-1.4 회원가입 성공

| 항목 | 내용 |
|------|------|
| [P] | 사용 가능한 닉네임 |
| [S] | 1. 회원가입 탭 → 2. 닉네임 입력 (사용가능 체크 표시 확인) → 3. 비밀번호 + 확인 입력 → 4. "가입하기" 클릭 → 5. 장르 선택 스텝에서 2~3개 선택 → 6. "시작하기" 클릭 |
| [E] | 닉네임 중복확인 디바운스 500ms 후 체크마크 표시, 비밀번호 강도 바 표시, 가입 후 장르 선택 화면 → `/personas` 리다이렉트 |

### TC-1.5 회원가입 실패 — 닉네임 중복

| 항목 | 내용 |
|------|------|
| [P] | 이미 등록된 닉네임 |
| [S] | 중복 닉네임 입력 → 디바운스 대기 |
| [E] | 닉네임 입력란 옆에 X 아이콘 + "이미 사용 중인 닉네임입니다" 표시, 가입 버튼 비활성화 |

### TC-1.6 회원가입 실패 — 비밀번호 불일치

| 항목 | 내용 |
|------|------|
| [P] | - |
| [S] | 비밀번호와 비밀번호 확인이 다르게 입력 |
| [E] | 가입 버튼 비활성화 또는 에러 메시지 표시 |

### TC-1.7 비밀번호 강도 표시

| 항목 | 내용 |
|------|------|
| [P] | 회원가입 탭 |
| [S] | 1. `abc` 입력 → 강도 "약함" 확인 → 2. `Abc12345` 입력 → "보통" 확인 → 3. `Abc1234!@#` 입력 → "강함" 확인 |
| [E] | 3단계 강도 바가 입력에 따라 색상/레벨 변화 |

### TC-1.8 비밀번호 표시/숨김 토글

| 항목 | 내용 |
|------|------|
| [P] | 로그인 또는 회원가입 탭 |
| [S] | 비밀번호 입력 후 눈 아이콘 클릭 |
| [E] | `type="password"` ↔ `type="text"` 토글, 입력값 유지 |

### TC-1.9 관리자 로그인 시 리다이렉트

| 항목 | 내용 |
|------|------|
| [P] | role=`admin` 또는 `superadmin` 계정 |
| [S] | 관리자 계정으로 로그인 |
| [E] | `/admin` 페이지로 리다이렉트 |

### TC-1.10 미인증 사용자 보호 라우트 접근

| 항목 | 내용 |
|------|------|
| [P] | 로그아웃 상태 |
| [S] | 브라우저 주소창에 `/personas` 직접 입력 |
| [E] | `/` 로그인 페이지로 리다이렉트 |

## 2. 페르소나 탐색 (홈)

**URL:** `/personas`
**API:** `GET /personas`, `GET /personas/tags`, `GET /favorites`, `POST /chat/sessions`, `POST/DELETE /favorites/{id}`, `POST /character-cards/import`

### TC-2.1 페이지 초기 로드

| 항목 | 내용 |
|------|------|
| [P] | 로그인 상태 |
| [S] | `/personas` 접근 |
| [E] | 환영 메시지에 닉네임 표시, 퀵 액세스 4개 카드, 검색바, 필터칩, 페르소나 카드 그리드 표시. API 3개 병렬 호출 (personas, tags, favorites) |

### TC-2.2 검색 — 키워드 필터링

| 항목 | 내용 |
|------|------|
| [P] | 페르소나가 2개 이상 등록 |
| [S] | 검색바에 특정 페르소나 이름 일부 입력 → 300ms 대기 |
| [E] | 디바운스 후 해당 키워드가 포함된 페르소나만 표시. `GET /personas?search=키워드` 호출 확인 |

### TC-2.3 연령등급 필터

| 항목 | 내용 |
|------|------|
| [P] | 전연령 + 15+ + 18+ 페르소나 각 1개 이상 등록 |
| [S] | 1. "전체" 칩 → 모든 등급 표시 → 2. "전연령" 칩 → all 등급만 → 3. "15+" → 15+ 등급만 → 4. "18+" → 18+ 등급만 |
| [E] | 각 필터에 맞는 페르소나만 표시. 미성인인증 사용자는 18+ 페르소나 카드 표시되지 않음 |

### TC-2.4 정렬 변경

| 항목 | 내용 |
|------|------|
| [P] | 페르소나 3개 이상 |
| [S] | 정렬 드롭다운에서 "인기순" → "이름순" → "최신순" 전환 |
| [E] | 각 정렬 기준에 따라 카드 순서 변경. `sort=popular|name|recent` 파라미터 확인 |

### TC-2.5 태그 필터

| 항목 | 내용 |
|------|------|
| [P] | 태그가 설정된 페르소나 존재 |
| [S] | 인기 태그 칩 중 하나 클릭 |
| [E] | 해당 태그를 가진 페르소나만 필터링. 태그 칩 활성화 스타일 변경 |

### TC-2.6 즐겨찾기 토글

| 항목 | 내용 |
|------|------|
| [P] | 로그인 상태, 페르소나 카드 표시 |
| [S] | 1. 하트 버튼 클릭 (즐겨찾기 추가) → 2. 같은 하트 버튼 다시 클릭 (해제) |
| [E] | 추가 시: 하트 채워짐 + like_count +1, `POST /favorites/{id}`. 해제 시: 하트 비워짐 + like_count -1, `DELETE /favorites/{id}` |

### TC-2.7 대화 시작

| 항목 | 내용 |
|------|------|
| [P] | 전연령 페르소나 존재 |
| [S] | 페르소나 카드의 "대화하기" 버튼 클릭 |
| [E] | `POST /chat/sessions` 호출 → `/chat/{sessionId}` 리다이렉트 |

### TC-2.8 18+ 페르소나 접근 — 미인증 사용자

| 항목 | 내용 |
|------|------|
| [P] | 성인인증 미완료 사용자, 18+ 페르소나 존재 |
| [S] | 18+ 페르소나 "대화하기" 클릭 |
| [E] | 403 에러 + 성인인증 유도 토스트 또는 모달 표시 |

### TC-2.9 캐릭터 카드 가져오기 (Import)

| 항목 | 내용 |
|------|------|
| [P] | 유효한 캐릭터 카드 JSON 파일 준비 (`{"name":"테스트","description":"...","personality":"..."}`) |
| [S] | "가져오기" 버튼 → 파일 선택 → JSON 파일 업로드 |
| [E] | 성공 토스트 "캐릭터 카드를 가져왔습니다", 목록 새로고침 후 새 페르소나 표시 |

### TC-2.10 캐릭터 카드 가져오기 실패 — 잘못된 JSON

| 항목 | 내용 |
|------|------|
| [P] | name 필드 없는 JSON 파일 |
| [S] | "가져오기" 버튼 → 잘못된 파일 업로드 |
| [E] | "유효하지 않은 캐릭터 카드: name 필드가 필요합니다" 에러 토스트 |

### TC-2.11 퀵 액세스 카드 네비게이션

| 항목 | 내용 |
|------|------|
| [P] | 로그인 상태 |
| [S] | "내 대화" / "챗봇 만들기" / "캐릭터 라운지" / "구독 & 크레딧" 각각 클릭 |
| [E] | `/sessions` / `/personas/create` / `/community` / `/mypage?tab=subscription` 정확히 이동 |

## 3. 페르소나 생성

| 항목 | 내용 |

| [E] | 403 에러, "성인인증이 필요합니다" 에러 토스트 |

### TC-3.3 18+ 페르소나 생성 — 성인인증 완료

| 항목 | 내용 |
|------|------|
| [P] | 성인인증 완료 사용자 |

### TC-3.4 필수 필드 미입력

| 항목 | 내용 |
|------|------|
| [P] | - |
| [S] | 캐릭터 이름 비워둔 채 "생성" 클릭 |
| [E] | 유효성 검사 에러 표시 (필수 필드 강조) |

### TC-3.5 선택 필드 입력 (description, greeting, scenario, tags)

| 항목 | 내용 |
|------|------|
| [P] | - |

| [E] | 모든 필드가 DB에 정상 저장. 이후 편집 화면에서 입력값 확인 |

|------|------|
