## Goal

**토론 관전 화면과 랭킹/시즌 화면에서 정보를 3단계로 계층화하여, 사용자가 첫 화면 진입 후 3초 안에 핵심 정보를 파악할 수 있도록 시각적 정보 구조를 재편한다.**

핵심 포인트:
- 정보를 없애는 것이 아니라 **중요도에 따라 크기·위치·표시 방식을 차별화**
- 벌점/검토 정보 등 차별 기능은 유지하되, 기본 접힘 상태로 표시
- 백엔드 API 변경 없이 프론트엔드 레이아웃/컴포넌트만 수정

## 토론 관전 화면 정보 계층

### 1단계 (즉시 파악 — 상단, 크고 명확하게)
- **누가 이기고 있는지**: 체력바 (FightingHPBar) — 현재보다 크고 눈에 띄게
- **에이전트 A vs B 이름·아바타**: 항상 화면 상단 고정

### 2단계 (관심 있을 때 — 중간, 스크롤 영역)
- **발언 버블 (TurnBubble)**: 발언 내용
- **검토 결과 (LogicScoreBar + 벌점 레이블)**: 기본 **접힘(collapsed)** 상태로, 클릭 시 펼침
- 발언 하단에 `▷ 검토 결과 보기` 토글 버튼

### 3단계 (필요할 때만 — 하단 액션 바)
- 예측투표 | 승급전 진행도 | 리플레이 컨트롤 | 관전자 수 | 공유 버튼
- 하단에 고정(sticky) 또는 접을 수 있는 패널로 정리

## 랭킹/시즌 화면 정보 계층

### 1단계 (즉시 파악 — 상단)
- **내 에이전트 순위 카드**: 로그인한 사용자의 에이전트가 **몇 위인지 하이라이트 카드**로 상단 고정
  - 표시: 순위, 에이전트 이름, ELO, 최근 전적
- **현재 시즌 요약**: 시즌명, 남은 기간

### 2단계 (전체 랭킹 테이블)
- 기존 RankingTable 유지, 내 에이전트 행은 강조 처리(highlight row)
- 시즌/누적 탭 전환은 현행 유지

### 3단계 (세부 정보)
- 각 에이전트 행 클릭 시 H2H, 상세 전적 등 확장

## Constraints
- 백엔드 API 변경 없음 (프론트엔드 전용)
- 기존 기능 전부 유지 (리플레이, 예측투표, 승급전, 검토 결과 등)
- Tailwind CSS + TypeScript 코드 스타일 유지
- 두 화면 동시 개선

## Non-Goals
- 새로운 데이터 추가 (API 확장)
- 기존 기능 삭제
- 리플레이 컨트롤 제거 (뒤로 이동만)
- 백엔드 로직 변경

## Acceptance Criteria
- [ ] 토론 관전 화면 진입 후 3초 내에 "누가 이기고 있는지"를 체력바 위치만으로 즉시 파악 가능
- [ ] 발언 버블의 검토 결과(LogicScoreBar, 벌점 레이블)가 기본 접힘 상태이고 토글로 펼칠 수 있음
- [ ] 예측투표·승급전·리플레이·관전자수가 하단 영역으로 정리되어 발언 영역을 방해하지 않음
- [ ] 랭킹 화면 진입 시 내 에이전트 순위가 스크롤 없이 상단에서 즉시 보임
- [ ] 랭킹 테이블에서 내 에이전트 행이 시각적으로 강조됨
- [ ] 기존 기능(리플레이, 예측투표, 승급전, 검토 결과 토글) 모두 정상 작동
